01 / Overview

What Is This Portfolio?

This is a personal portfolio website built for Aritra Das — a final-year Computer Science student. The goal is to showcase projects, skills, experience and contact information in a visually stunning, animated, and professional way — just like a resume but on the internet.

🌐 What you see when you visit the website:
  • A cinematic loading splash screen
  • An animated hero section with your name, typewriter text, and floating profile photo
  • Sections: About → Skills → Projects → Experience → Education → Activities → Contact
  • Interactive project cards with mouse-tracking glow
  • A resume modal (PDF viewer popup)
  • Matrix rain background animation
  • Scroll progress bar, back-to-top rocket button

How Does index.html Connect to React?

index.html has a single <div id="root">. React takes over this div and renders everything inside it. The script tag in index.html loads main.jsx:

<!-- index.html -->
<div id="root"></div>
<script type="module" src="/src/main.jsx"></script>

// main.jsx — mounts the App component into the #root div
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

ReactDOM.createRoot(document.getElementById('root')).render(<App />)
04 / React Components

What Is a React Component?

A component is a reusable piece of UI — like a function that returns HTML. Instead of writing one giant HTML file, we split the page into small, focused parts.

Components used in this portfolio:
SplashScreen

The cinematic intro screen shown on first load. Types out > INITIALIZING PORTFOLIO... then fades out.

Cursor

Replaces the default browser cursor with a glowing custom red dot + trailing ring.

Particles

Generates 22 floating colored dots that drift upward in the hero background.

MatrixRain

Draws falling characters (A-Z, 0-9, Japanese katakana) on an HTML Canvas element.

Typewriter

Cycles through phrases like "AI/ML Engineer", "Flask Developer" — types and deletes them in a loop.

FadeInVisible

Wraps any content and fades it in when it scrolls into the viewport using IntersectionObserver.

CountUp

Animates numbers from 0 to their target value (e.g. 0 → 8.70 CGPA) when scrolled into view.

ScrollProgress

The thin glowing bar at the top of the page that fills as you scroll down.

BackToTop

The 🚀 rocket button that appears after scrolling 400px and returns you to the top.

ProjectCard

Each project card with mouse-tracking glow effect, tech stack tags, and GitHub/Live links.

App (main)

The root component — assembles all sections (Hero, About, Skills, Projects, etc.) and manages state.

05 / Key Concepts

Key React Concepts Used

useState — Managing Data That Changes

When something on the page can change (like which project filter is active, or whether the resume modal is open), we use useState.

const [filter, setFilter] = useState('all');
// filter = current value ('all', 'ai', 'web', 'java')
// setFilter = function to change it

// When you click "AI / ML" button:
setFilter('ai');  // React re-renders the card list showing only AI projects

useEffect — Running Code At The Right Time

Code that should run after the component loads (like starting an animation, adding an event listener) goes inside useEffect.

useEffect(() => {
  // This runs once when the component first appears on screen
  window.addEventListener('scroll', handleScroll);

  return () => {
    // This runs when the component is removed — cleanup
    window.removeEventListener('scroll', handleScroll);
  };
}, []); // Empty [] means "run once on mount"

useRef — Grabbing a Real DOM Element

When you need to directly control an HTML element (like a canvas or an input), use useRef.

const canvasRef = useRef(null);
// Later, in useEffect:
const canvas = canvasRef.current; // Gets the real <canvas> element
const ctx = canvas.getContext('2d'); // Draw on it!

IntersectionObserver — Detecting Scroll Position

Used in FadeInVisible and CountUp. It watches an element and fires a callback when it enters the visible area of the screen.

const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('vis'); // Add class to trigger CSS animation
    }
  });
}, { threshold: 0.1 }); // Trigger when 10% of element is visible

observer.observe(someElement);
06 / Animations

How Do The Animations Work?

CSS Keyframe Animations

Most visual animations are pure CSS. You define a @keyframes block describing start → end states, then apply it to an element.

/* Floating profile photo */
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50%       { transform: translateY(-12px); }
}
.hero-photo { animation: float 5s ease-in-out infinite; }
/* The photo gently bobs up and down every 5 seconds forever */

/* Scroll progress bar */
@keyframes underline-grow {
  from { transform: scaleX(0); }
  to   { transform: scaleX(1); }
}

Matrix Rain — HTML Canvas

The matrix rain is drawn using the Canvas 2D API — a low-level drawing surface in the browser. Every 60ms, we:

1. Fill canvas with a semi-transparent dark overlay (creates "fading trail" effect)
2. Pick a random character from: A-Z, 0-9, Japanese katakana
3. Draw it at a random column position, moving downward
4. When a column reaches the bottom, randomly reset it to the top

Typewriter Effect

A JavaScript setInterval loop adds/removes one character at a time from the displayed text, cycling through phrases:

phrases = ['AI / ML Engineer', 'Flask Developer', 'JARVIS Builder', ...]
// Add 1 character every 78ms → looks like typing
// Remove 1 character every 42ms → looks like deleting (faster)
// After deleting, move to next phrase and repeat

Custom Cursor

The browser's default cursor is hidden (cursor: none in CSS). Two <div> elements are positioned with JavaScript to follow the mouse — one snaps instantly (small dot), one follows with a smooth lag (ring).

// Main dot snaps to cursor position
curRef.current.style.left = mouseX + 'px';
curRef.current.style.top  = mouseY + 'px';

// Ring uses lerp (linear interpolation) for smooth lag
ringX += (mouseX - ringX) * 0.12; // Moves 12% closer every frame
ringRef.current.style.left = ringX + 'px';

3D Photo Hover Effect

On hover, the About section photo gets a CSS 3D transform and a scanline laser effect:

.about-photo:hover {
  transform: scale(1.03) rotateY(-5deg) rotateX(5deg); /* 3D tilt */
  box-shadow: 0 30px 60px rgba(255,45,85,.3);           /* Neon glow */
}

/* Green laser scanning from top to bottom */
@keyframes scanline {
  0%   { top: -30%; }
  100% { top: 120%; }
}
07 / CSS Design System

How Is The Design System Built?

CSS Custom Properties (Variables)

All colors are defined once as variables in :root, so changing one value updates the whole site.

:root {
  --r:    #FF2D55;  /* Red — primary accent */
  --r2:   #FF6B35;  /* Red-orange — gradient end */
  --b:    #0066FF;  /* Blue */
  --b2:   #00D4FF;  /* Cyan */
  --p:    #BF00FF;  /* Purple */
  --g:    #00FF88;  /* Green */
  --y:    #FFD700;  /* Gold */
  --dark: #06060F;  /* Background */
  --card: #0F0F22;  /* Card background */
  --text: #F0F0FF;  /* Main text */
  --muted:#6060A0;  /* Dimmed text */
}

Glassmorphism

The frosted glass look on the navbar is achieved with:

nav {
  background: rgba(6, 6, 15, 0.88);  /* Semi-transparent dark */
  backdrop-filter: blur(28px);        /* Blur everything behind it */
  border-bottom: 1px solid rgba(255,45,85,0.15); /* Subtle red border */
}

Gradient Text

The name "DAS" and other gradient text uses CSS clip:

.hn2 {
  background: linear-gradient(135deg, #FF2D55, #FF6B35, #FFD700);
  -webkit-background-clip: text;       /* Clip gradient to text shape */
  -webkit-text-fill-color: transparent; /* Hide actual text color */
}

09 / Build & Deploy

How Does Build & Deployment Work?

Development (local)

npm run dev
→ Vite starts a local server at http://localhost:5173
→ Any file you save auto-refreshes the browser (Hot Module Replacement)
→ Fast — no page reload needed

Production Build

npm run build
→ Vite bundles all .jsx + .css files into optimized .js and .css
→ Output goes to portfolio/dist/
→ dist/index.html  ← The final single HTML file
→ dist/assets/index.js   ← All React code, minified
→ dist/assets/index.css  ← All CSS, minified

GitHub → Vercel Pipeline

You push code to GitHub
       ↓
Vercel detects the new commit automatically
       ↓
Vercel runs: npm install → npm run build
       ↓
Vercel takes the dist/ folder and serves it globally via CDN
       ↓
Your portfolio is live at yoursite.vercel.app within ~60 seconds
💡 vercel.json inside portfolio/ tells Vercel: use npm run build as build command and serve the output from the dist folder.
